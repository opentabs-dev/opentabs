OpenTabs gives AI agents access to any web application through your browser. No API keys, no OAuth, no service accounts — just your existing logged-in sessions.

Your AI assistant connects to an MCP server that dispatches tool calls to a Chrome extension. The extension runs plugin code directly in the page, with full access to the DOM, cookies, and session storage. If you can see it in your browser, your AI agent can interact with it.

## How It Works

<HowItWorks />

1. **You start the MCP server** — it discovers installed plugins and exposes their tools to any MCP client (Claude Code, Cursor, etc.).

2. **The Chrome extension connects** — it receives plugin definitions from the server and watches for matching tabs. When you open a tab that matches a plugin's URL pattern, the extension injects the plugin's adapter into the page.

3. **Your AI agent calls a tool** — the server dispatches the call through the extension to the right tab. The plugin code runs in the page context with your authenticated session, and the result flows back to the agent.

## Key Features

- **Zero-configuration access** — uses your existing browser sessions. No API keys, no OAuth, no service accounts.
- **Plugin ecosystem** — anyone can create and publish OpenTabs plugins as standalone npm packages. Install with `npm install -g` and they're auto-discovered.
- **Built-in browser tools** — tab management, screenshots, DOM interaction, network capture, cookies, console logs, and more — available for every tab, no plugin required.
- **Works with any MCP client** — Claude Code, Cursor, and any client that supports Streamable HTTP transport.
- **Hot reload for developers** — the MCP server detects plugin changes automatically. Zero-restart development workflow.

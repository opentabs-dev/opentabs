## npm Authentication (One-Time Setup)

Create a granular access token at [npmjs.com/settings/tokens/create](https://www.npmjs.com/settings/tokens/create):

- **Permissions**: Read and Write
- **Packages**: `@opentabs-dev/*`
- **Bypass 2FA**: Enabled

Save the token to `~/.npmrc`:

```bash
echo '//registry.npmjs.org/:_authToken=<YOUR_TOKEN>' > ~/.npmrc
```

Verify it works:

```bash
npm whoami
```

## Publishing a Release

The `scripts/publish.sh` script handles the entire publish workflow:

```bash
./scripts/publish.sh 0.0.17
```

The script:

1. Verifies npm authentication via `npm whoami`
2. Runs `bun run build` to compile all packages
3. Bumps `"version"` in all four `platform/*/package.json` files
4. Updates cross-references between packages (e.g., `plugin-sdk` depends on `shared`)
5. Runs `bun run build` again with the new versions
6. Publishes in dependency order: `shared` â†’ `plugin-sdk` â†’ `plugin-tools` â†’ `cli`

All packages use `"publishConfig": { "access": "restricted" }` â€” only `@opentabs-dev` org members can install them.

## After Publishing

After publishing new platform package versions, update plugin dependencies:

```bash
cd plugins/slack
# Update @opentabs-dev/* versions in package.json
bun install
bun run build
```

The `opentabs-plugin build` command embeds the installed SDK version as `sdkVersion` in `dist/tools.json`. The MCP server checks this at discovery time â€” if a plugin was built with a newer SDK than the server, it is rejected with a clear rebuild message.

## Version Compatibility

The MCP server uses major.minor comparison for SDK version checks:

- A plugin's major.minor must be **less than or equal to** the server's major.minor
- Patch differences are always fine
- Plugins without `sdkVersion` in `tools.json` (predating this feature) load with a warning

This means you can publish a patch release (e.g., `0.0.16` â†’ `0.0.17`) and existing plugins continue to work without rebuilding. A minor or major bump (e.g., `0.0.x` â†’ `0.1.0`) requires plugins to be rebuilt with the new SDK.

## What NOT to Publish

- **`platform/mcp-server`** â€” ships as a bundled runtime via the CLI
- **`platform/browser-extension`** â€” installed to `~/.opentabs/extension/` by the CLI
- **`platform/create-plugin`** â€” published separately as `create-opentabs-plugin` (not part of the `@opentabs-dev` scope)
- **`plugins/*`** â€” example plugins are standalone projects that publish independently to npm under `opentabs-plugin-*` names

## For Users

Use OpenTabs with existing plugins and the built-in browser tools. You need **Node.js 20+** and **Google Chrome** (or any Chromium-based browser).

### Install the CLI

<CliCommand npmCommand="npm install -g @opentabs-dev/cli" />

### Start the server

```bash
opentabs start
```

On first run, this creates the `~/.opentabs/` directory, generates an auth secret, and installs the Chrome extension files.

### Load the Chrome extension

1. Open `chrome://extensions/` in Chrome
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked** and select `~/.opentabs/extension`

The extension connects to the server automatically.

### Configure your MCP client

Get your auth secret:

```bash
opentabs config show --json | jq -r .secret
```

Add the server to your MCP client. For Claude Code (`~/.claude.json` — merge into `"mcpServers"`):

```json
{
  "mcpServers": {
    "opentabs": {
      "type": "streamable-http",
      "url": "http://localhost:9515/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_SECRET_HERE"
      }
    }
  }
}
```

Replace `YOUR_SECRET_HERE` with the secret from the previous command.

### Install plugins

```bash
opentabs plugin search
npm install -g opentabs-plugin-slack
opentabs start
```

Restart the server after installing new plugins so it discovers them.

### Verify

```bash
opentabs status
opentabs doctor
```

You're ready. Your AI agent can now use all registered plugin tools and the 37 built-in browser tools.

## For Plugin Developers

Build your own OpenTabs plugins. You need everything in **For Users** above.

### Set up the user environment

Follow the [For Users](#for-users) section above first. You need a working OpenTabs installation to test your plugins.

### Scaffold a new plugin

<CliCommand npmCommand="npx create-opentabs-plugin my-plugin" />

```bash
cd my-plugin
npm install
```

### Build and register

```bash
npm run build
```

This compiles your plugin, generates `dist/adapter.iife.js` and `dist/tools.json`, registers the plugin in `~/.opentabs/config.json` under `localPlugins`, and notifies the running server via `POST /reload`.

### Rebuild and reload

Every time you rebuild, the server picks up your changes automatically:

```bash
npm run build   # Compiles, builds adapter, and notifies the server via POST /reload
```

The `opentabs-plugin build` step (part of `npm run build`) calls `POST /reload` on the running server, triggering plugin rediscovery. No server restart needed.

See the [Plugin Development guide](/docs/guides/plugin-development) for a complete walkthrough of building your first plugin.

## For Contributors

Work on the OpenTabs platform itself. You need **[Bun](https://bun.sh)** (v1.3.9+), **[Node.js](https://nodejs.org)** (v20+), and **Google Chrome**.

### Clone and build

```bash
git clone https://github.com/opentabs-dev/opentabs.git
cd opentabs
bun install
bun run build
```

### Load the Chrome extension

1. Open `chrome://extensions/` in Chrome
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked** and select `~/.opentabs/extension`

### Start the dev server

```bash
bun --hot platform/mcp-server/dist/index.js --dev
```

This starts the MCP server with Bun's hot reload enabled (contributor-only). When you rebuild platform packages (`bun run build`), Bun re-evaluates modules without restarting the process.

### Configure your MCP client

Get the auth secret from the auto-generated config:

```bash
cat ~/.opentabs/config.json | jq -r .secret
```

Add it to your MCP client as shown in the [For Users](#configure-your-mcp-client) section.

### Run the checks

```bash
bun run build         # Build all packages
bun run type-check    # TypeScript check
bun run lint          # ESLint
bun run knip          # Unused code detection
bun run test          # Unit tests
bun run test:e2e      # E2E tests (Playwright)
```

All commands must exit 0 before committing. See the [Development Setup guide](/docs/contributing/dev-setup) for the full contributor workflow.

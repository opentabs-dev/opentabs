```bash
npm install -g @opentabs-dev/cli
```

## opentabs start

Start the MCP server in production mode.

```bash
opentabs start [options]
```

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Option</Table.Head>
      <Table.Head>Default</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`--port <number>`</Table.Cell>
      <Table.Cell>`9515`</Table.Cell>
      <Table.Cell>Server port.</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>

On first run, `opentabs start` auto-initializes the `~/.opentabs/` directory:

1. Creates `~/.opentabs/config.json` with a generated secret
2. Installs the Chrome extension files to `~/.opentabs/extension/`
3. Prints first-time setup instructions (load the extension, configure your MCP client)

On subsequent runs, the extension is updated if the CLI version has changed.

Server output is written to both the terminal and `~/.opentabs/server.log`. The process exits if the port is already in use.

```bash
# Start on default port
opentabs start

# Custom port
opentabs start --port 3000
```

## opentabs status

Show MCP server status and connected plugins.

```bash
opentabs status [options]
```

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Option</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`--port <number>`</Table.Cell>
      <Table.Cell>Server port to check (default: `9515`).</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`--json`</Table.Cell>
      <Table.Cell>Output raw JSON from the `/health` endpoint.</Table.Cell>
    </Table.Row>
  </Table.Body>

Displays server version, SDK version, uptime, extension connection, MCP client count, plugin count, total tool count, and per-plugin details (tab state, tool count, source).

Tab states are color-coded: ready (green), unavailable (yellow), closed (dim). Failed plugins are listed separately with their error messages.

```bash
opentabs status
opentabs status --json
opentabs status --port 3000
```

## opentabs audit

Show recent tool invocation history from the server's in-memory audit log.

```bash
opentabs audit [options]
```

      <Table.Cell>Filter by plugin name.</Table.Cell>
    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`--json`</Table.Cell>
      <Table.Cell>—</Table.Cell>

Each entry shows the timestamp, tool name, success/failure status, and duration. Failed entries include the error code, category, and message when using `--json`.

The server maintains the last 500 tool invocations in memory. The audit log resets when the server restarts.

```bash
# Show last 20 invocations
opentabs audit

# Show last 50, filtered to one plugin
opentabs audit --limit 50 --plugin slack

# Raw JSON output
opentabs audit --json
```

## opentabs doctor

Run diagnostic checks on your OpenTabs setup.

```bash
opentabs doctor [options]
```

<Table>
  <Table.Header>
    <Table.Row>
      <Table.Head>Option</Table.Head>
      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>
    <Table.Row>
      <Table.Cell className="font-medium">`--port <number>`</Table.Cell>
      <Table.Cell>Server port to check (default: `9515`).</Table.Cell>
    </Table.Row>
  </Table.Body>
</Table>

Performs the following checks:

1. **Bun runtime** — verifies Bun is installed
2. **Config file** — checks `~/.opentabs/config.json` exists
3. **MCP server health** — fetches the `/health` endpoint (3s timeout)
4. **Extension connection** — checks if the Chrome extension is connected via WebSocket
5. **Extension installed** — verifies `~/.opentabs/extension/manifest.json` exists
6. **Extension version** — compares installed extension version to CLI version
7. **Local plugins** — for each local plugin, checks that `dist/tools.json` and `dist/adapter.iife.js` exist

Results are shown as a checklist with pass (✓), warning (!), and error (✗) indicators. Exits with code 1 if any fatal check fails.

```bash
opentabs doctor
opentabs doctor --port 3000
```

```bash

```

    </Table.Row>
    <Table.Row>
      <Table.Cell className="font-medium">`--plugin <name>`</Table.Cell>

Reads from `~/.opentabs/server.log`. By default, follows the file for new output (like `tail -f`). The `--plugin` flag filters to lines containing `[plugin:<name>]`, showing only log entries from that plugin.

```bash
# Follow logs in real time
opentabs logs

# Show last 100 lines and exit
opentabs logs --lines 100 --no-follow

# Show only Slack plugin logs
opentabs logs --plugin slack
```

```bash
# Install using shorthand
opentabs plugin install slack

# Install using full package name
opentabs plugin install opentabs-plugin-slack

# Install a scoped community plugin
opentabs plugin install @my-org/opentabs-plugin-custom
```

## opentabs config

View and manage configuration. Has three subcommands: `show`, `set`, and `path`.

### opentabs config show

Display the current configuration (secret redacted).

```bash
opentabs config show [options]
```

<Table>
  <Table.Header>
    <Table.Row>

      <Table.Head>Description</Table.Head>
    </Table.Row>
  </Table.Header>
  <Table.Body>

      <Table.Cell>Output config as JSON (includes the actual secret value).</Table.Cell>
    </Table.Row>
  </Table.Body>

```bash
opentabs config show
opentabs config show --json

# Retrieve the secret for MCP client configuration
opentabs config show --json | jq -r .secret
```

### opentabs config set

Set a configuration value.

```bash
opentabs config set <key> <value>
```

Supports enabling and disabling individual tools:

```bash
# Disable a tool
opentabs config set tool.slack_send_message disabled

# Re-enable a tool
opentabs config set tool.slack_send_message enabled
```

The key format is `tool.<plugin>_<tool>` and value must be `enabled` or `disabled`.

### opentabs config path

Print the config file path.

```bash
opentabs config path
# Output: /Users/you/.opentabs/config.json
```
